    
Описание программы
---------------------------------

  Программа предназначена для обнаружения и графического отображения топологии L2 сети.
Работа программы состоит из следующих этапов:
1) Ввод пользователем информации о сети через GUI;
2) Запрос у коммутаторов в сети информации по SNMP;
3) Обработка полученной информации;
4) Графическое отображение топологии;

Основной принцип, заложенный в этап 3 - обнаружение однозначных тупиковых ветвей, с последующим продвижением к корню дерева топологии.


Краткое описание классов
--------------------------------

1) Bulk - посредством библиотеки snmp4j реализует Bulk-запрос на коммутатор (методы test и send), принимает ответ и преобразует
его в вид, удобный для дальнейшей обработки (метод onResponse).

2) GUI - использует библиотеку swing для создания окон пользовательского интерфейса, запрашивает количество устройств и их ip-адреса
у пользователя, проверяет введенные данные на корректность и передает их экземпляру класса ConnectionSearch.

3) Comparator - отправляет два вида Bulk запросов с разными oid (методы InterfaceDiscovery и MACDiscovery) на IP-адрес коммутатора,
метод TableComplete преобразует полученные данные в hashmap с записами вида MAC-интерфейс.

4) ConnectionSearch - создает экземпляры класса Comparator (количество вводится пользователем в GUI), каждый из которых возвращает таблицу 
MAC-интерфейс. Методы FindingNodes и RemoveNodes находят и убирают из таблиц записи с собственными маками коммутаторов; методы
FindingDirectConnections DeleteAlreadyKnownConnections находят и удаляют из таблиц тупиковые ветви, занося их в отдельную hashmap
(для каждого экземпляра Comparator - своя hashmap). Итого на выходе - n map со строками вида "mac устройства - mac соседа".

5) graph - По таблицам экземпляра ConnectionSearch строит граф топологии, через вершины которого можно вызвать информационное окно об узле.



Особенности и возможность доработки
------------------------------------

Из-за сжатых сроков дипломного проектирования не была решена проблема протоколом RSTP на коммутаторах MES2124/3124, поэтому тестирование
программы производилось при настроенной статической коммутации. Требует обнаружения, понимания и устранения.

Размер Bulk запроса ограничен 50 строками, неизвестно поведение при тестировании на более крупных сетях, нужно проработать разбитие данных
на несколько запросов.

Опять же из-за сжатых сроков в качестве ТЗ было большое количество вводимой пользователем информации, необходимо уменьшить взаимодействие
путем автоматического опроса сети (например пользователь вводит лишь пул адресов).

Вся обработка информации включает в себя форматирование, основанное на том виде, в котором возвращают данные коммутаторы MES3124/2124,
поскольку создавалась программа под них. Необходимы тесты на других вендорах с целью унификации.
